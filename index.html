项目总览

本系统整合「2014-2026年合同台账管理」与「物业合同审计比对」核心功能，实现从合同数据标准化处理到合规审计、风险管控、底稿归档的全流程覆盖，适配8000+份合同数据，支持多格式读取、跨年度统计与审计实操一体化。

📊 数据规模

  • 覆盖年度：2014-2026（13个年度）
  • 合同总量：8000+份
  • 支持格式：.xls（2014-2022）、.xlsx（2023-2026）
  • 核心字段：合同编号、金额、日期、经办部门、类型、归档状态等

🔧 核心功能模块

  • 台账管理：多格式读取、数据清洗、跨年度统计
  • 合规审计：制度条款比对、违规操作排查
  • 价格管控：市场对标分析、异常价格识别
  • 风险管控：风险点管理、整改跟踪
  • 报告归档：自动生成审计报告、底稿分类导出
  • 模糊查询：合同编号/客方名称快速检索

📈 核心价值

  • 一体化操作：台账管理与审计功能无缝衔接
  • 专项适配：覆盖台风灾后、疫情免租等10+年度场景
  • 合规闭环：从核查到整改、归档全流程管控
  • 轻量高效：无复杂依赖，本地运行保障数据安全

快速开始

1. 克隆仓库

git clone https://github.com/你的用户名/兆翔物业多年度合同管理系统.git
cd 兆翔物业多年度合同管理系统

2. 安装依赖

pip install pandas openpyxl numpy xlrd

注：xlrd用于支持2014-2022年.xls格式台账，openpyxl支持2023-2026年.xlsx格式

3. 运行代码

将目标年度合同台账文件放入项目根目录，修改主程序配置后执行：

# 在main.py中配置目标年度与文件路径
TARGET_YEARS = [2014, 2020, 2023]  # 支持单个/多个年度
FILE_PATHS = [
    "2014年合同台帐.xls",
    "2020年合同台账.xlsx",
    "2023年合同台账.xlsx"
]

# 执行主程序
python main.py

4. 核心操作流程

  1. 导入合同数据后，通过「年度台账特征」页面的模糊查询快速定位目标合同；
  2. 进入「审计比对功能」完成合规核查与价格异常筛选；
  3. 通过「违规风险管理」跟踪问题整改；
  4. 最后通过「底稿归档导出」生成标准化归档文件。

制度合规比对核查（排查物业人员操作是否合规）

左侧粘贴公司制度条款（审计基准），右侧粘贴合同/审批流程等审计底稿，自动比对识别不合规的违规操作
